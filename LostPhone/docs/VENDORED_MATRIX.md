# Matrice vendoring — 61 apps tier Awesome

Objectif : remplacer les vues Awesome générées par des **clones GitHub SwiftUI vendored** (recette zerocode117).

**Légende effort**
- **S** — copy-paste, peu ou pas d’adaptation (< 20 fichiers, pas de Firebase)
- **M** — vendoring + strip backend / données demo
- **L** — gros repo ou Firebase + refonte données
- **XL** — pas de clone SwiftUI crédible → commission (Anukar) ou garder Awesome

**Statut**
- `live` — dans `VendoredShowroomCatalog`
- `candidate` — repo identifié, à vendorer
- `flutter` — clone Flutter seulement (conversion = L)
- `none` — rien trouvé
- `awesome` — garder Awesome en fallback

## Workflow par app

```bash
# 1. Cloner le candidat
git clone --depth 1 <repo-url> /tmp/vendor-<app>

# 2. Copier vers Apps/Vendored/<App>/
# 3. Créer LpspVendored<App>RootView.swift
# 4. Rédiger SOURCE.md
# 5. Ajouter dans VendoredShowroomCatalog + VendoredShowroomRouter
# 6. Tester showroom → app
```
